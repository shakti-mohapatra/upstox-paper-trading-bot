## 1. COST MODEL — EXACT UPSTOX MATH (NOT ESTIMATES)

Your previous calculation used `Round-trip cost ≈ ₹2` for RELIANCE qty=1. **That is wrong.** Here is the verified Upstox cost structure (as of 2026):

### Upstox Intraday Equity Charges (per order, NSE)

| Charge | Rate | Side |
|--------|------|------|
| **Brokerage** | ₹20 or 0.1% whichever is **lower** | Both Buy & Sell |
| **STT** | 0.025% | Sell only |
| **Exchange Transaction** | 0.00297% | Both |
| **Stamp Duty** | 0.003% | Buy only |
| **SEBI** | ₹10/crore (0.0001%) | Both |
| **IPFT** | ₹0.10/lakh (0.0001%) | Both |
| **GST** | 18% on (Brokerage + Transaction + IPFT) | Both |
| **Auto Square-off** | ₹75 + GST | If triggered by broker |

### Verified Round-Trip Cost Formula (Code This Into Your Bot)

```python
def calculate_costs(turnover):
    # Calculate total round-trip cost for intraday equity trade on Upstox (NSE)
    # turnover = entry_price * qty (same for exit, assuming same price level)

    # Buy side costs
    brokerage_buy = min(20, turnover * 0.001)
    txn_buy = turnover * 0.0000297
    stamp_buy = turnover * 0.00003
    sebi_buy = turnover * 0.000001
    ipft_buy = turnover * 0.000001
    gst_buy = 0.18 * (brokerage_buy + txn_buy + ipft_buy)
    buy_cost = brokerage_buy + txn_buy + stamp_buy + sebi_buy + ipft_buy + gst_buy

    # Sell side costs
    brokerage_sell = min(20, turnover * 0.001)
    txn_sell = turnover * 0.0000297
    stt_sell = turnover * 0.00025
    sebi_sell = turnover * 0.000001
    ipft_sell = turnover * 0.000001
    gst_sell = 0.18 * (brokerage_sell + txn_sell + ipft_sell)
    sell_cost = brokerage_sell + txn_sell + stt_sell + sebi_sell + ipft_sell + gst_sell

    return buy_cost + sell_cost  # Total round-trip
```

## 2. BREAKEVEN MATH — THE REAL NUMBERS

Use this formula in your bot's pre-trade filter:

```
Net Win = (Gross Target %) - (Cost %)
Net Loss = (Gross Stop %) + (Cost %)
Breakeven Win Rate = Net Loss / (Net Win + Net Loss)
```

### Cost-Adjusted Breakeven Win Rates (₹1L Turnover = 0.083% Cost)

## 3. POSITION SIZING — FIXED FRACTIONAL (NOT FIXED RUPEE)

Risk percentage is NOT profit margin. It is the maximum capital you are willing to lose if stopped out.

### Rule: Risk 1.0% of Capital Per Trade (Maximum)

```python
def position_size(capital, entry_price, stop_price):
    # Fixed fractional position sizing.
    # Returns 0 if trade is not viable due to cost constraints.

    risk_amount = capital * 0.01  # 1% of capital
    risk_per_share = abs(entry_price - stop_price)

    if risk_per_share <= 0:
        return 0

    qty = int(risk_amount / risk_per_share)

    # Constraint: Max 20% of capital in one trade
    max_turnover = capital * 0.20
    max_qty_by_turnover = int(max_turnover / entry_price)
    qty = min(qty, max_qty_by_turnover)

    # Minimum viable: ensure costs don't eat >10% of potential win
    # At 0.083% cost, we need turnover >= ~₹10K for cost efficiency
    min_turnover = 10000
    if qty * entry_price < min_turnover:
        return 0  # Skip trade — too small, costs will kill edge

    return qty
```

### Sizing Examples (₹1L Capital)

| Entry | Stop | Risk/Share | Qty | Turnover | Capital Used | Risk |
|-------|------|------------|-----|----------|--------------|------|
| ₹1000 | ₹995 (0.5%) | ₹5 | 20 | ₹20,000 | 20% | ₹100 (1.0%) |
| ₹500 | ₹495 (1.0%) | ₹5 | 20 | ₹10,000 | 10% | ₹100 (1.0%) |
| ₹2000 | ₹1990 (0.5%) | ₹10 | 10 | ₹20,000 | 20% | ₹100 (1.0%) |
| ₹500 | ₹497.5 (0.5%) | ₹2.5 | 40 | ₹20,000 | 20% | ₹100 (1.0%) |

> **Key Point:** With ₹1L capital and 1% risk, you can trade ₹10K–₹20K notional. This keeps cost % near the minimum (~0.083%).

## 5. STRATEGY SELECTION LOGIC — THE BOT'S "BRAIN"

The bot must **not** run all strategies simultaneously. Use this decision tree at 9:15 AM:

```python
def select_strategy():
    # Select ONE strategy for the day based on market conditions.
    # Called once at 9:15 AM. No switching mid-day.

    # Nifty 50 pre-market / first 5 min data
    gap = (nifty_open - nifty_prev_close) / nifty_prev_close

    # ADX of Nifty 50 on daily timeframe
    adx = get_nifty_daily_adx()

    if abs(gap) > 1.5 or adx > 30:
        return "MOMENTUM_BREAKOUT"  # Strong trend day expected
    elif abs(gap) > 0.5:
        return "ORB_15MIN"  # Volatile open expected
    elif adx < 20:
        return "VWAP_MEAN_REVERSION"  # Range-bound day expected
    else:
        return "EMA_CROSSOVER"  # Trend building
```

> **🚨 BOT RULE:** Only **one** strategy per day. No mixing. No "if Strategy A fails, try Strategy B." Commit at 9:15 and stick to it.
